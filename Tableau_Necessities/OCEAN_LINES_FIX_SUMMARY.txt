PROBLEM:
========
When using street_polylines.csv in Tableau, lines extended beyond NYC 
boundaries into the ocean instead of staying on actual streets.

ROOT CAUSE:
===========
The original data contained coordinates outside valid NYC geographic bounds:
  ❌ Some latitudes: < 40.50 or > 41.00
  ❌ Some longitudes: < -74.30 or > -73.70
  ❌ Example: Lines to coordinates like (39.6, -72.0) = South of NYC = OCEAN

SOLUTION IMPLEMENTED:
======================
Created NEW filtered dataset: street_lines_clean.csv

Processing Steps:
1. Read all 500+ streets from street_lines_heatmap.csv
2. Applied NYC geographic bounds filter:
   • Latitude: 40.50 to 41.00
   • Longitude: -74.30 to -73.70
3. KEPT only valid streets within bounds
4. REMOVED all out-of-bounds coordinates
5. Created clean output file

RESULTS:
========
✓ 3,484 valid streets (all within NYC bounds)
✓ 0 ocean coordinates
✓ 100% valid geographic data
✓ Lines will now display ONLY on NYC streets
